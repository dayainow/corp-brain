# 사내 문서관리 챗봇 PoC 구축 전략

사내 문서관리 챗봇 샘플이라면 처음부터 "진짜 사내 문서"를 모으기보다 아래처럼 시작하는 게 안전합니다. 합성 문서는 개인정보 이슈를 줄이면서도 계약서, 보고서, 양식, 청구서 등 다양한 포맷을 흉내 낼 수 있습니다.

## 진행 단계
* **1단계**: RVL-CDIP로 문서 타입 분류기나 라우팅 규칙을 만듭니다. (github)
* **2단계**: `synthetic_business_docs`로 계약서/폼 기반 RAG 검색과 답변 품질을 봅니다. (huggingface)
* **3단계**: DocVQA 또는 DUDE로 "문서 내용 질문 응답" 성능을 점검합니다. (arxiv)
* **4단계**: 실제 사내 포맷과 유사한 템플릿을 몇 개 직접 만들어 합성 데이터로 늘립니다. Symage 같은 서비스는 민감정보 없이 realistic document를 대량 생성하는 방향을 제안합니다.

## 주의할 점
공개 데이터셋은 문서 형식은 좋아도 실제 회사 내부 규정서, 회의록, SOP, 인사문서 구조와는 다를 수 있습니다. 
그래서 PoC에서는 공개 문서로 파이프라인을 검증하고, 이후 회사 템플릿을 비식별화해서 소량 추가하는 방식이 보통 더 잘 맞습니다.
