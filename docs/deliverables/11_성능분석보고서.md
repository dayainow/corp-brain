| 영역 | 현재 수준 (2026-07-03) | 목표 |

| 검색 Hit@3 | **100.0%** (25쿼리 eval) | **≥ 80%** ✓ |

| 단위 테스트 | 79건 PASS | 유지 |

## 3. 응답 지연 (Latency)

### 3.1 채팅 요청 구간

| 단계 | 예상 지연 | 병목 |
|------|-----------|------|
| 임베딩 생성 | 200~800ms | Transformers.js CPU, 최초 모델 로드 수 초 |
| hybridSearch | 10~100ms (JSON) / 5~50ms (PgVector ANN) | 저장소·청크 수 |
| UIMessage data parts | 검색·생성 단계 이벤트 | `data-rag-status`, `data-rag-sources` |
| Ollama TTFT | 1~5s | GPU/CPU, 모델 크기 |
| 스트리밍 전체 | 5~30s | `maxDuration=30` |

### 3.2 인덱싱

| 모드 | vault 22파일 기준 | 비고 |
|------|-------------------|------|
| full | ~1~3분 | 임베딩 per chunk (~96 chunks) |
| incremental | 변경 파일만 | manifest hash 비교 |

### 3.3 Admin Metrics API

- 전체 eval 재실행: 25쿼리 × (임베딩 + 검색)
- **15분 캐시** — `?refresh=true`로 강제 갱신
