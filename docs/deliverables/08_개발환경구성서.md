## 2. 클론 후 포함·미포함

### Git에 포함

| 항목 | 경로·비고 |
|------|-----------|
| 앱 소스 | `src/` |
| 샘플 Vault | `vault/` 22종 문서 |
| 시드 계정 | `src/lib/auth/users.ts` (PoC 비밀번호 `novapay2026`) |
| Compose | `docker-compose.yml`, `scripts/deploy-compose.sh` |
| eval·테스트 | `data/eval-queries.json`, `e2e/`, `*.test.ts` |

### Git에 없음 (클론 후 생성)

| 항목 | 생성 |
|------|------|
| `.env.local` | `cp .env.example .env.local` + `AUTH_SECRET` |
| 벡터 인덱스 | `npm run index:vault` 또는 Admin Sync Vault |
| `vectors.json` | 인덱싱 시 생성 (`.gitignore`) |

### 클론한 사람 필수 준비

- Node.js 20+, npm, Git
- Ollama + `llama3` (`ollama run llama3`)
- `AUTH_SECRET` 32자 이상

Slack·Google SSO·웹훅은 **웹 UI만** 사용 시 생략 가능.

## 3. 초기 설정

```bash
# 1. 저장소 클론
git clone https://github.com/dayainow/corp-brain.git
cd corp-brain

# 2. 의존성 설치
npm ci

# 3. 환경 변수
cp .env.example .env.local
```

### `.env.local` 필수 항목

```bash
AUTH_SECRET=<32자 이상 랜덤>   # openssl rand -base64 32
AUTH_URL=http://localhost:3000
VAULT_PATH=./vault
EMBEDDING_MODEL=Xenova/multilingual-e5-small
```

### Ollama

```bash
# 별도 터미널
ollama run llama3
```

## 4. 실행 명령

| 명령 | 용도 |
|------|------|
| `npm run dev` | 개발 서버 (`:3000`) |

## 6. Docker Compose (PgVector)

```bash
./scripts/deploy-compose.sh   # 또는 npm run deploy:compose
# 호스트 PgVector 인덱싱: config/env/compose.host.env 참고
```

### docker-compose 서비스

| 서비스 | 포트 | 설명 |
|--------|------|------|
| postgres | **5433**→5432 | pgvector (Mac 로컬 PG 충돌 회피) |
| redis | 6379 | Rate limit |
| app | **3100**→3000 | Next.js standalone (호스트 접속) |

Ollama(`llama3`)는 **호스트** `:11434`에서 별도 실행.

## 10. 트러블슈팅

| 증상 | 해결 |
|------|------|
| 채팅 무응답 | Ollama 실행 확인 |
| Sync 실패 | `VAULT_PATH=./vault` |
| Health degraded | Sync Vault 실행 |
| E2E 포트 충돌 | `:3001` — `playwright.config` reuseExistingServer (로컬) |
| 임베딩 느림 | 최초 1회 HF 모델 다운로드 대기 |
