## 1. 개요

NovaPay 임직원이 사내 문서를 권한에 맞게 검색·질의할 수 있는 로컬 RAG 챗봇의 기능 요구사항을 정의합니다.

---

## 2. 기능 요구사항 목록

### FR-001 인증·권한

| ID | 요구사항 | 우선순위 | 수용 기준 |
|----|----------|----------|-----------|
| FR-001-01 | NextAuth 기반 로그인 | 필수 | Credentials·Google SSO 지원 |
| FR-001-02 | Role 기반 접근 제어 | 필수 | general / manager / admin 3단계 |
| FR-001-03 | 미인증 접근 차단 | 필수 | `/login` 리다이렉트 |
| FR-001-04 | Admin 전용 화면 보호 | 필수 | non-admin → `/` 리다이렉트 |

### FR-002 RAG 채팅

| ID | 요구사항 | 우선순위 | 수용 기준 |
|----|----------|----------|-----------|
| FR-002-01 | 사내 문서 기반 답변 | 필수 | Context 외 추측 금지 프롬프트 |
| FR-002-02 | 스트리밍 응답 | 필수 | 토큰 단위 UI 갱신 |
| FR-002-03 | 출처 인용 | 필수 | `[출처: filename.md]` 형식 |
| FR-002-04 | 한국어 답변 | 필수 | 시스템 프롬프트 한국어 강제 |
| FR-002-05 | 멀티턴 검색 | 권장 | 후속 질문 맥락 반영 (`buildSearchQuery`) |
| FR-002-06 | 답변 피드백 | 권장 | 👍/👎 → 감사 로그 |

### FR-003 문서·인덱싱

| ID | 요구사항 | 우선순위 | 수용 기준 |
|----|----------|----------|-----------|
| FR-003-01 | Vault 문서 인덱싱 | 필수 | `.md` `.pdf` `.docx` 지원 |
| FR-003-02 | RBAC Pre-filtering | 필수 | frontmatter `role` 기반 |
| FR-003-03 | 전체 Sync Vault | 필수 | admin 전용 |
| FR-003-04 | 증분 Sync | 권장 | mtime/hash manifest |
| FR-003-05 | 웹 업로드 | 필수 | manager+, 5MB, role 지정 |
| FR-003-06 | 문서 만료 | 권장 | `expires` 만료 시 검색 제외 |

### FR-004 검색 품질

| ID | 요구사항 | 우선순위 | 수용 기준 |
|----|----------|----------|-----------|
| FR-004-01 | Hybrid 검색 | 필수 | Vector + Keyword + RRF |
| FR-004-02 | Re-ranking | 필수 | 파일명·제목·한국어 토큰 가산 |
| FR-004-03 | 검색 품질 평가 | 권장 | Hit@3, MRR (`eval:search`) |
| FR-004-04 | PgVector ANN | 권장 | 운영 환경 DB 레벨 검색 |

### FR-005 운영·보안

| ID | 요구사항 | 우선순위 | 수용 기준 |
|----|----------|----------|-----------|

### FR-006 Admin·품질

## 3. 비기능 요구사항

| ID | 항목 | 요구 | 현재 |
|----|------|------|------|
| NFR-001 | 데이터 유출 방지 | LLM·임베딩 로컬 처리 | Ollama + Transformers.js |
| NFR-002 | RBAC 위반 | 0% | 단위·E2E 검증 |
| NFR-003 | Hit@3 | ≥ 80% (목표) | eval CI ≥ 50% |
| NFR-004 | P50 응답 | ≤ 5초 (목표) | Ollama 의존 |
| NFR-005 | 가용성 | Health degraded 분리 | vault·인덱스·Ollama |
