## 3. TypeScript

- `strict: true` 유지
- `any` 사용 지양 — `unknown` + 타입 가드
- 공개 API·함수 시그니처에 명시적 반환 타입 권장
- enum 대신 `as const` + union type 선호 (`UserRole` 등)

```typescript
// 권장: 타입 가드
export function isValidUserRole(role: string): role is UserRole {
  return USER_ROLES.includes(role as UserRole);
}
```

---

## 4. React·Next.js

- **Client Component**: `"use client"` — 상호작용·훅 필요 시만
- **Server Component**: 기본값, 데이터 fetch·정적 UI
- 페이지별 상태는 해당 `page.tsx`에 두고, 재사용 UI만 `components/`로 분리
- `useChat` 등 AI SDK 훅은 Client Component에서만 사용

---

## 5. API Route

- 인증: `requireAuth(minimumRole?)` (`lib/auth/guard.ts`)
- Rate limit: `checkRateLimit` + `denyRateLimit`
- 에러 응답: `{ error, code }` JSON + 적절한 HTTP status
- 감사: 민감 작업은 `writeAuditLog` 호출
- 로깅: `logError(scope, { err, path })`

```typescript
const { error, session } = await requireAuth();
if (error) return error;
```

## 7. 테스트

### 7.1 단위 테스트 (Vitest)

- 위치: 소스와 동일 디렉터리 `*.test.ts`
- 범위: RBAC, 메시지 변환, 검색 메트릭, 한국어 쿼리 등 순수 로직
- 실행: `npm test`

### 7.2 E2E (Playwright)

- 위치: `e2e/*.spec.ts`
- 포트 **3001** 격리 서버 (`playwright.config.ts`)
- 실행: `npm run test:e2e` (32 시나리오)
- RAG: `npm run test:e2e:rag` (Ollama + index:vault)

### 7.3 품질 하네스

- 5팀 에이전트: `.ax/harnesses/`
- 실행: `npm run harness:quality`
- 전체 루프: `npm run quality:loop`

### 7.4 검색 품질 게이트

- `npm run eval:search` — Hit@3 ≥ `EVAL_HIT3_THRESHOLD` (기본 **80%**, CI)
- 파일럿: `npm run pilot:ready`, `npm run report:pilot-weekly`
- 인덱싱 선행: `npm run index:vault`
