## 1. 전체 사용자 흐름

```mermaid
flowchart TD
    Start([접속]) --> Auth{로그인?}
    Auth -->|No| Login["/login"]
    Login --> Cred[이메일·비밀번호 또는 Google SSO]
    Cred --> AuthOK{인증 성공?}
    AuthOK -->|No| Login
    AuthOK -->|Yes| Chat["/ 채팅"]
    Auth -->|Yes| Chat

    Chat --> Q[질문 입력]
    Q --> API["POST /api/chat"]
    API --> Ans[스트리밍 답변 + 출처]
    Ans --> FB{피드백?}
    FB -->|👍/👎| AuditF[audit: chat.feedback]
    FB -->|계속| Q

    Chat --> Help[도움말 패널]
    Chat --> Guide["/guide"]
    Chat --> Upload{manager+?}
    Upload -->|Yes| UpModal[문서 업로드]
    UpModal --> Index1[증분 인덱싱]

    Chat --> Sync{admin?}
    Sync -->|Yes| SyncBtn[Sync Vault]
    SyncBtn --> Index2[전체/증분 인덱싱]

    Chat --> AdminLink{admin?}
    AdminLink -->|Yes| Admin["/admin"]
    Admin --> Logs[감사 로그·메트릭]
```

## 2. 인증 흐름

```mermaid
sequenceDiagram
    participant U as 사용자
    participant B as Browser
    participant M as Middleware
    participant N as NextAuth
    participant API as API Route

    U->>B: / 접속
    B->>M: 요청
    alt 미로그인
        M-->>B: 302 /login
        U->>B: 로그인 폼 제출
        B->>N: Credentials / Google
        N-->>B: JWT 세션 (role 포함)
    end
    B->>API: POST /api/chat
    API->>API: requireAuth()
    API-->>B: 스트리밍 응답
```

## 3. RAG 질의 흐름

```mermaid
sequenceDiagram
    participant UI as 채팅 UI
    participant Chat as /api/chat
    participant RAG as lib/chat/rag
    participant VS as VectorStore
    participant O as Ollama

    UI->>Chat: messages[]
    Chat->>RAG: buildSearchQuery + retrieveRagContext
    RAG->>VS: hybridSearch (RBAC)
    VS-->>RAG: Top-K chunks
    RAG->>O: streamText(system + context)
    O-->>UI: SSE 스트리밍
    Chat->>Chat: audit chat.query
```

## 4. 문서 인덱싱 흐름

```mermaid
flowchart LR
    A[Admin: Sync Vault] --> B{mode}
    B -->|full| C[runIndexing]
    B -->|incremental| D[runIncrementalSync]
    C --> E[vault/ 재귀 스캔]
    D --> F[manifest 비교]
    E --> G[청킹 + 임베딩]
    F --> G
    G --> H[(VectorStore)]
    H --> I[검색 가능]
```
