|------|------|

### 2.2 Role 계층

| Role | 설명 | 문서 접근 |
|------|------|-----------|
| `general` | 일반 직원 | general 문서 |
| `manager` | 팀장·실무 책임 | general + manager |
| `admin` | 운영·보안 | 전체 |

```typescript

canReindexVault(userRole)           // admin
```

### 2.3 API 가드

`requireAuth(minimumRole?)` — 미인증 401, 권한 부족 403

### 2.4 Slack Role 매핑

- `SLACK_USER_MAP` env: `U12345:manager,U67890:admin`
- `resolveSlackUserRole(userId)` — 미매핑 시 `general`

## 5. 벡터 저장·검색 (검색품질팀)

### 5.1 VectorStore 인터페이스

| 구현 | env | 설명 |
|------|-----|------|
| `JsonVectorStore` | `VECTOR_STORE=json` (기본) | `src/data/vectors.json` |
| `PgVectorStore` | `VECTOR_STORE=pgvector` | PostgreSQL + pgvector ANN |

### 5.2 Hybrid Search 파이프라인

`hybridSearch(query, embedding, topK, userRole)` — `lib/vector-store/index.ts`

```
fetchSearchCandidates (ANN + ILIKE)
  → 만료 문서 필터 (isDocumentExpired)
  → reciprocalRankFusion (벡터·키워드 RRF)
  → rerankCandidates (한국어 동의어·토큰 가중)
  → topK 반환
```

### 5.3 한국어 검색 보강

| 모듈 | 함수 | 역할 |
|------|------|------|
| `korean-query.ts` | `normalizeKoreanQuery` | 공백·조사 정규화 |
| `korean-query.ts` | `expandKoreanTokens` | 동의어 확장 |
| `query-context.ts` | `buildSearchQuery` | 멀티턴 맥락 병합 |
| `reranker.ts` | `rerankCandidates` | 2차 스코어링 |

### 5.4 PgVector ANN

`PgVectorStore.fetchSearchCandidates` — cosine distance `<=>` + ILIKE fallback

## 6. RAG 채팅 (RAG팀)

### 6.1 메시지 파이프라인

| 단계 | 함수 | 파일 |
|------|------|------|
| 검증 | `validateChatMessages` | `lib/chat/messages.ts` |
| 변환 | `toModelMessages` | 동일 |
| 텍스트 추출 | `extractMessageText` | 동일 (`parts` undefined 방어) |
| 검색 쿼리 | `buildSearchQuery` | `lib/search/query-context.ts` |

### 6.2 RAG 컨텍스트

`retrieveRagContext(searchQuery, latestMessage, userRole)` — `lib/chat/rag.ts`

1. `generateEmbedding(searchQuery)`
2. `hybridSearch` (topK = `RAG_TOP_K`, 기본 5)
3. Context 문자열 + `buildSystemPrompt` (한국어·출처 인용 규칙)

### 6.3 스트리밍 응답 (UIMessage Stream)

**서버** (`src/app/api/chat/route.ts`)

```
createUIMessageStream
  ├─ writer: data-rag-status { phase: "searching" }   (transient)
  ├─ retrieveRagContext(searchQuery, …)
  ├─ writer: data-rag-status { phase: "generating" }  (transient)
  ├─ writer: data-rag-sources { sources: buildRagSourceCards(docs) }
  └─ writer.merge(streamRagResponse → toUIMessageStream)
```

**클라이언트** (`src/app/page.tsx`)

| 훅/상태 | 역할 |
|---------|------|
| `useChat` + `DefaultChatTransport` | POST `/api/chat` 스트리밍 |
| `onData` | `data-rag-status` → `ragPhase` |
| `extractRagSourcesFromParts` | assistant `parts`에서 출처 카드 |
| `buildSourceHighlightMap` | 원문 모달 청크 하이라이트 |

`streamRagResponse` → AI SDK `streamText` + Ollama (`llama3`)

응답 헤더: `Content-Type: text/event-stream`, Rate limit `X-RateLimit-*`

### 6.4 후속 질문 칩 (클라이언트)

| 항목 | 내용 |
|------|------|
| 함수 | `suggestFollowUpQuestions(query, answer, sources)` |
| 파일 | `lib/chat/follow-up-suggestions.ts` |
| UI | `FollowUpChips` — 마지막 assistant 답변 완료 시 최대 3개 |
| 전송 | `sendMessage({ text })` — 서버 API 변경 없음 |

### 6.5 피드백

- UI: `ChatFeedback` 컴포넌트
- API: `POST /api/chat/feedback` — audit `chat.feedback`

## 10. 감사·헬스 (플랫폼팀)

### 9.1 감사 로그

- 파일: `data/audit.log` (JSON Lines)
- SIEM: `AUDIT_WEBHOOK_URL` 설정 시 `exportToSiem`
- Admin 조회: `GET /api/admin/audit`

### 10.2 Health

`GET /api/health` — vault, chunk 수, Ollama, vector store, **postgres**(pgvector), **redis**

| status | 조건 |
|--------|------|
| ok | 모든 check 통과, chunk > 0 |
| degraded | chunk 0 또는 부분 실패 |
| unhealthy | 치명적 실패 → 503 |
